# DialogForm 组件使用规范文档

## 一、组件概述

`DialogForm` 是一个基于 Naive UI 的表单对话框组件，支持平铺模式和分组模式，提供丰富的表单字段类型和验证功能。

**组件路径**: `@/components/DialogForm.vue`

## 二、基础使用

### 2.1 导入组件

```vue
import DialogForm from '@/components/DialogForm.vue'
```

### 2.2 基础模板结构

```vue
<DialogForm
    ref="dialogRef"
    dialogPreset="card"
    v-model:visible="showDialog"
    v-model:formData="formData"
    :use-field-groups="true"
    :field-groups="fieldGroups"
    :rules="formRules"
    :title="dialogTitle"
    :positive-text="dialogType === 'add' ? '创建' : '更新'"
    :validate-on-submit="true"
    :show-success-message="true"
    success-message="保存成功！"
    @submit="handleSubmit"
    @field-change="handleFieldChange"
>
  <!-- 自定义底部操作按钮 -->
  <template #action="{ formData }">
    <n-space justify="end">
      <n-button size="small" @click="handleCancel">取消</n-button>
      <n-button
          size="small"
          type="success"
          :loading="submitting"
          @click="handleSubmit(formData, true)"
      >
        确定
      </n-button>
    </n-space>
  </template>
</DialogForm>
```

## 三、Props 配置

### 3.1 基础属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `visible` | Boolean | - | 对话框显示状态（v-model） |
| `type` | String | - | 对话框类型 |
| `title` | String | - | 对话框标题 |
| `positiveText` | String | '确认' | 确认按钮文字 |
| `negativeText` | String | '取消' | 取消按钮文字 |

### 3.2 表单数据

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `formData` | Object | {} | 表单数据对象（v-model） |
| `rules` | Object/Function | {} | 验证规则，支持函数形式接收当前表单数据 |

### 3.3 字段配置

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `fields` | Array | [] | 字段配置数组（平铺模式） |
| `fieldGroups` | Array | [] | 字段分组配置数组（分组模式） |
| `useFieldGroups` | Boolean | false | 是否使用分组模式 |

### 3.4 表单配置

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `labelPlacement` | String | 'left' | 标签位置 |
| `labelWidth` | String/Number | 'auto' | 标签宽度 |
| `labelAlign` | String | 'left' | 标签对齐方式 |
| `size` | String | 'medium' | 表单尺寸 |
| `disabled` | Boolean | false | 是否禁用 |

### 3.5 对话框配置

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `dialogPreset` | String | 'dialog' | 对话框预设类型（推荐用 'card'） |
| `showIcon` | Boolean | true | 是否显示图标 |
| `maskClosable` | Boolean | true | 点击遮罩是否关闭 |
| `closable` | Boolean | true | 是否显示关闭按钮 |

### 3.6 提交配置

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `validateOnSubmit` | Boolean | true | 提交时是否验证 |
| `showSuccessMessage` | Boolean | false | 是否显示成功消息 |
| `successMessage` | String | '操作成功' | 成功消息内容 |
| `loading` | Boolean | false | 是否加载中 |

## 六、验证规则

### 6.1 基础验证

```javascript
const formRules = {
  username: [
    {required: true, message: '请输入用户名', trigger: ['blur', 'input']},
    {min: 3, max: 20, message: '长度在3-20个字符', trigger: ['blur', 'input']}
  ],
  email: [
    {required: true, message: '请输入邮箱', trigger: 'blur'},
    {type: 'email', message: '邮箱格式不正确', trigger: 'blur'}
  ]
}
```

### 6.2 动态验证（推荐）

```javascript
const formRules = (model) => ({
  password: [
    {required: true, message: '请输入密码', trigger: 'blur'},
    {min: 6, message: '密码至少6位', trigger: 'blur'}
  ],
  confirmPassword: [
    {required: true, message: '请确认密码', trigger: 'blur'},
    {
      validator: (rule, value) => {
        if (value !== model.password) {
          return new Error('两次输入的密码不一致')
        }
        return true
      },
      trigger: ['blur']
    }
  ],
  nodeId: [
    {
      validator: (rule, value) => {
        if (value && !model.path) {
          return new Error('选择节点后必须填写路径')
        }
        return true
      },
      trigger: ['blur']
    }
  ]
})
```

### 6.3 常用验证规则

| 规则 | 说明 |
|------|------|
| `{required: true}` | 必填 |
| `{type: 'email'}` | 邮箱格式 |
| `{type: 'number'}` | 数字类型 |
| `{min: 3, max: 20}` | 长度范围 |
| `{pattern: /regex/}` | 正则表达式 |
| `validator` | 自定义验证函数 |

## 七、完整使用示例

### 7.1 基础代码结构

```vue
<template>
  <n-card title="示例页面">
    <n-button type="primary" @click="handleAdd">新建</n-button>

    <DialogForm
        ref="dialogRef"
        dialogPreset="card"
        v-model:visible="showDialog"
        v-model:formData="formData"
        :use-field-groups="true"
        :field-groups="fieldGroups"
        :rules="formRules"
        :title="dialogTitle"
        :positive-text="dialogType === 'add' ? '创建' : '更新'"
        :validate-on-submit="true"
        :show-success-message="true"
        success-message="保存成功！"
        @submit="handleSubmit"
        @field-change="handleFieldChange"
    >
      <template #action="{ formData }">
        <n-space justify="end">
          <n-button size="small" @click="handleCancel">取消</n-button>
          <n-button
              size="small"
              type="success"
              :loading="submitting"
              @click="handleSubmit(formData, true)"
          >
            确定
          </n-button>
        </n-space>
      </template>
    </DialogForm>
  </n-card>
</template>

<script setup>
import {ref, computed} from 'vue'
import DialogForm from '@/components/DialogForm.vue'

// 状态定义
const showDialog = ref(false)
const submitting = ref(false)
const dialogType = ref('add')
const dialogRef = ref(null)

// 表单数据
const formData = ref({
  name: '',
  email: '',
  status: 1,
  enabled: true
})

const defaultFormData = {
  name: '',
  email: '',
  status: 1,
  enabled: true
}

// 字段分组
const fieldGroups = computed(() => [
  {
    title: '基本信息',
    description: '请填写基础信息',
    fields: [
      {
        name: 'name',
        label: '名称',
        type: 'input',
        placeholder: '请输入名称',
        required: true,
        maxlength: 50
      },
      {
        name: 'email',
        label: '邮箱',
        type: 'input',
        placeholder: '请输入邮箱',
        required: true
      }
    ]
  },
  {
    title: '配置',
    fields: [
      {
        name: 'status',
        label: '状态',
        type: 'select',
        placeholder: '请选择状态',
        options: [
          {label: '启用', value: 1},
          {label: '禁用', value: 0}
        ]
      },
      {
        name: 'enabled',
        label: '启用',
        type: 'switch',
        checkedValue: true,
        uncheckedValue: false
      }
    ]
  }
])

// 验证规则
const formRules = (model) => ({
  name: [
    {required: true, message: '请输入名称', trigger: ['blur', 'input']},
    {min: 2, max: 50, message: '长度在2-50个字符', trigger: ['blur', 'input']}
  ],
  email: [
    {required: true, message: '请输入邮箱', trigger: 'blur'},
    {type: 'email', message: '邮箱格式不正确', trigger: 'blur'}
  ]
})

// 计算属性
const dialogTitle = computed(() => {
  return dialogType.value === 'add' ? '新建' : '编辑'
})

// 方法
const handleAdd = () => {
  dialogType.value = 'add'
  formData.value = {...defaultFormData}
  showDialog.value = true
}

const handleEdit = (row) => {
  dialogType.value = 'edit'
  formData.value = {...row}
  showDialog.value = true
}

const handleCancel = () => {
  showDialog.value = false
  formData.value = {...defaultFormData}
}

const handleSubmit = async (data, validate = false) => {
  if (validate && dialogRef.value) {
    try {
      await dialogRef.value.validate()
    } catch (error) {
      console.log('表单验证失败:', error)
      return
    }
  }

  formData.value = {...data}
  submitting.value = true

  try {
    if (dialogType.value === 'add') {
      await window.$request.post('/api/resource', formData.value)
    } else {
      await window.$request.put(`/api/resource/${formData.value.id}`, formData.value)
    }
    window.$message.success('保存成功')
    showDialog.value = false
    // 刷新列表...
  } catch (error) {
    window.$message.error('保存失败')
  } finally {
    submitting.value = false
  }
}

const handleFieldChange = ({fieldName, value}) => {
  console.log(`字段 ${fieldName} 变化:`, value)
}
</script>
```

## 八、最佳实践

### 8.1 数据处理

- 使用 `computed` 配置 `fieldGroups`，实现动态选项
- 使用 `...` 展开运算符避免引用问题
- 使用 `defaultFormData` 初始化表单

### 8.2 验证规则

- 优先使用函数形式的验证规则，实现字段间联动验证
- 使用 `validator` 处理复杂验证逻辑

### 8.3 事件处理

- 使用 `@field-change` 监听字段变化
- 在 `handleSubmit` 中先验证再提交

### 8.4 API 调用

- 使用 `window.$request` 统一请求
- 错误信息使用 `error.response?.data?.message`

### 8.5 UI 优化

- 使用 `dialogPreset="card"` 获得更好的视觉效果
- 自定义 `#action` 插槽控制按钮样式
- 使用 `loading` 状态防止重复提交

### 11.2 window.$request - 请求工具

**功能**: 封装的 axios 实例，自动处理认证、错误提示和主题适配

**基础用法**:

```javascript
// GET 请求
const data = await window.$request.get('/api/resource')
const data = await window.$request.get('/api/resource', { params: { page: 1 } })

// POST 请求
const result = await window.$request.post('/api/resource', { name: 'test' })

// PUT 请求
const result = await window.$request.put('/api/resource/1', { name: 'updated' })

// DELETE 请求
await window.$request.delete('/api/resource/1')

// 导出文件
await window.$request.exportFile('/api/export', { id: 1 }, 'filename.json')
```

**特性**:
- 自动添加 Authorization Bearer Token（认证路径除外）
- 自动处理 401/403 认证错误
- 统一错误提示
- 自动返回 `response.data.data`（code=200 时）
- 支持文件导出功能

**公共路径**（无需认证）:
- `/sys/init/check`
- `/sys/init/setup`
- `/sys/login`
- `/sys/health`

**错误处理**:
```javascript
try {
  const result = await window.$request.post('/api/resource', formData)
  // 成功：result 即为 data
} catch (error) {
  // 失败：已自动显示错误提示，无需手动处理
  console.error('请求失败:', error)
}
```
