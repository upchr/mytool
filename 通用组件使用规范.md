## 一、DialogForm 组件概述

`DialogForm` 是一个基于 Naive UI 的表单对话框组件，支持平铺模式和分组模式，提供丰富的表单字段类型和验证功能。

**组件路径**: `@/components/DialogForm.vue`

## 二、基础使用

### 2.1 导入组件

```vue
import DialogForm from '@/components/DialogForm.vue'
```

### 2.2 基础模板结构

```vue
<DialogForm
    ref="dialogRef"
    dialogPreset="card"
    v-model:visible="showDialog"
    v-model:formData="formData"
    :use-field-groups="true"
    :field-groups="fieldGroups"
    :rules="formRules"
    :title="dialogTitle"
    :positive-text="dialogType === 'add' ? '创建' : '更新'"
    :validate-on-submit="true"
    :show-success-message="true"
    success-message="保存成功！"
    @submit="handleSubmit"
    @field-change="handleFieldChange"
>
  <!-- 自定义底部操作按钮 -->
  <template #action="{ formData }">
    <n-space justify="end">
      <n-button size="small" @click="handleCancel">取消</n-button>
      <n-button
          size="small"
          type="success"
          :loading="submitting"
          @click="handleSubmit(formData, true)"
      >
        确定
      </n-button>
    </n-space>
  </template>
</DialogForm>
```

### 3.1 核心属性

| **提交** | `validateOnSubmit` | Boolean | true | 提交时验证 |

### 4.2 字段配置结构

```javascript
  placeholder: '请输入...', // 占位符
  required: true,           // 是否必填

```

### 4.3 常用配置速查

| 场景 | 配置示例 |
|------|----------|
| 必填文本 | `{name: 'username', type: 'input', required: true, maxlength: 50}` |
| 多行文本 | `{name: 'desc', type: 'textarea', autosize: {minRows: 2, maxRows: 4}}` |
| 下拉选择 | `{name: 'status', type: 'select', options: [{label: '启用', value: 1}]}` |
| 开关 | `{name: 'enabled', type: 'switch', checkedValue: true, uncheckedValue: false}` |
| 文件上传 | `{name: 'file', type: 'upload', action: '/api/upload', accept: '.jpg,.png'}` |

**注意**：文件上传的 `action` 属性必须包含 `/api/` 前缀，因为它直接用于 `<n-upload>` 组件，不经过 `window.$request`。

## 六、验证规则

### 6.1 基础验证

```javascript
const formRules = {
  username: [
    {required: true, message: '请输入用户名', trigger: ['blur', 'input']},
    {min: 3, max: 20, message: '长度在3-20个字符'}
  ]
}
```

### 6.2 动态验证（推荐）

```javascript
const formRules = (model) => ({
  password: [
    {required: true, message: '请输入密码'}
  ],
  confirmPassword: [
    {
      validator: (rule, value) => {
        return value === model.password ? true : new Error('两次密码不一致')
      },
      trigger: ['blur']
    }
  ]
})
```

### 6.3 常用验证规则

| 规则 | 说明 |
|------|------|
| `{required: true}` | 必填 |
| `{type: 'email'}` | 邮箱格式 |
| `{min: 3, max: 20}` | 长度范围 |
| `{pattern: /regex/}` | 正则表达式 |
| `validator` | 自定义验证函数 |

## 七、最佳实践

- **动态选项**：使用 `computed` 配置 `fieldGroups`
- **避免引用问题**：使用 `...` 展开运算符
- **字段联动验证**：使用函数形式的验证规则
- **API 调用**：使用 `window.$request` 统一请求
- **UI 优化**：`dialogPreset="card"` 获得更好效果

## ⚠️ 重要：dialogPreset 使用规范

### 7.1 dialogPreset="card"（推荐）

**特点：**
- 卡片样式，视觉效果更好
- 不会显示默认操作按钮
- 必须使用 `#action` 插槽自定义按钮

**正确用法：**
```vue
<DialogForm
    ref="dialogRef"
    dialogPreset="card"
    v-model:visible="showDialog"
    v-model:formData="formData"
    :field-groups="fieldGroups"
    :rules="formRules"
    title="表单标题"
    :validate-on-submit="true"
    @submit="handleSubmit"
>
  <template #action="{ formData }">
    <n-space justify="end">
      <n-button size="small" type="default" @click="handleCancel">取消</n-button>
      <n-button size="small" type="success" :loading="submitting" @click="handleSubmit(formData, true)">确定</n-button>
    </n-space>
  </template>
</DialogForm>
```

**提交方法实现：**
```javascript
const handleSubmit = async (data, needValidate = false) => {
  // 如果需要手动验证
  if (needValidate && dialogRef.value) {
    try {
      await dialogRef.value.validate()
      console.log('✅ 表单验证通过')
    } catch (error) {
      console.log('❌ 表单验证失败:', error)
      return // 验证失败，不提交
    }
  }

  // 提交逻辑
  submitting.value = true
  try {
    await window.$request.post('/api/resource', data)
    window.$message.success('保存成功')
    showDialog.value = false
  } finally {
    submitting.value = false
  }
}
```

### 7.2 dialogPreset="dialog"（默认）

**特点：**
- 对话框样式
- 自动显示"确定"、"取消"按钮
- 点击"确定"时组件会自动验证表单

**正确用法：**
```vue
<DialogForm
    ref="dialogRef"
    dialogPreset="dialog"
    v-model:visible="showDialog"
    v-model:formData="formData"
    :field-groups="fieldGroups"
    :rules="formRules"
    title="表单标题"
    :validate-on-submit="true"
    @submit="handleSubmit"
/>
```

**提交方法实现：**
```javascript
const handleSubmit = async (data) => {
  submitting.value = true
  try {
    await window.$request.post('/api/resource', data)
    window.$message.success('保存成功')
    showDialog.value = false
  } finally {
    submitting.value = false
  }
}
```

### 7.3 使用建议

| 场景 | 推荐配置 |
|------|----------|
| 简单表单 | `dialogPreset="dialog"`（简单，无需自定义按钮） |
| 复杂表单 | `dialogPreset="card"`（美观，可自定义按钮） |
| 需要自定义按钮逻辑 | `dialogPreset="card"` + `#action` 插槽 |
| 需要显示加载状态 | `dialogPreset="card"` + `#action` 插槽 |

### 7.4 常见错误

❌ **错误：使用 `dialogPreset="card"` 但没有 `#action` 插槽**
```vue
<DialogForm
    dialogPreset="card"
    v-model:visible="showDialog"
    v-model:formData="formData"
    @submit="handleSubmit"
/>
<!-- ❌ 没有显示操作按钮，无法提交表单 -->
```

✅ **正确：使用 `dialogPreset="card"` 并提供 `#action` 插槽**
```vue
<DialogForm
    dialogPreset="card"
    v-model:visible="showDialog"
    v-model:formData="formData"
    @submit="handleSubmit"
>
  <template #action="{ formData }">
    <n-button @click="handleCancel">取消</n-button>
    <n-button @click="handleSubmit(formData, true)">确定</n-button>
  </template>
</DialogForm>
```

## 九、常见问题

| 问题 | 解决方案 |
|------|----------|
| 字段联动验证 | 使用函数形式的 `formRules(model)` |
| 动态显示字段 | 使用 `visible: (formData) => ...` |
| 文件上传 | `{name: 'file', type: 'upload', action: '/api/upload'}` |
| 自定义验证消息 | 在规则中使用 `message` 属性 |
| API 路径错误 | 使用 `window.$request` 时不要添加 `/api/` 前缀，如 `/resource`；但文件上传的 `action` 必须包含 `/api/` 前缀 |
| **dialogPreset="card" 时没有显示操作按钮** | 必须使用 `#action` 插槽自定义按钮，参考本文档"⚠️ 重要：dialogPreset 使用规范"章节 |
| **使用 #action 插槽后验证失败** | 在提交方法中调用 `handleSubmit(formData, true)`，第二个参数 `true` 表示需要手动验证 |
| **dialogPreset="dialog" 时无法自定义按钮** | 改用 `dialogPreset="card"` 并提供 `#action` 插槽 |

## 十、全局方法

### 10.1 window.$request - 请求工具

#### 路由转发机制

项目使用 Nginx 反向代理，`request.js` 的 `baseURL` 配置为 `/api`，Nginx 会自动将 `/api/` 前缀的请求转发到后端并抹除前缀：

```
前端调用              → Nginx 转发                → 后端接收
/resource            → /api/ → http://127.0.0.1:8000/ → /resource
                     → 抹除 /api/ 前缀
```

**重要：**
- ✅ **正确**：直接使用后端路由路径，如 `/resource`
- ❌ **错误**：不要添加 `/api/` 前缀，如 `/api/resource`（会导致 `/api/api/resource`）

#### 基础用法

```javascript
// 正确用法：直接使用后端路由路径
const data = await window.$request.get('/resource', { params: { page: 1 } })
const result = await window.$request.post('/resource', { name: 'test' })
await window.$request.delete('/resource/1')
await window.$request.exportFile('/export', { id: 1 }, 'filename.json')
```

**特性**：自动认证、错误提示、返回 `response.data.data`
